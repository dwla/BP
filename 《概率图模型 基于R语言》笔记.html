第一章 概率推理

概率图模型（Probabilistic Graphical Models, PGM）：你想描述不同变量之间的关系，但是，你又对这些变量不太确定，只有一定程度的相信或者一些不确定的知识。概率，是表示和处理不确定性的严密数学方法。图，指图论，带有边和点的数学对象。概率图模型是使用概率来表示关于事实和事件的信念和不确定知识的一种工具。

机器学习，是关于从数据和观察中学习和适应的算法研究，并实现推理和借助学到的模型和算法来执行任务。

贝叶斯派思想的主要优势，是它不需要长期频率或者同一个试验的重复。

机器学习尤其是概率图模型的核心是条件概率的思想，这种概率的思想可以完美解释改变直觉认识或者给定新的信息来更新信念。

# 先验分布、似然率和数据序列
prior <- c(working = 0.99, broken = 0.01)
likelihood <- rbind(
  working = c(good = 0.99, bad = 0.01),
  broken = c(good = 0.6, bad = 0.4))
data <- c("bad", "bad", "bad", "bad")

# 贝叶斯更新函数
bayes <- function(prior, likelihood, data) {
  posterior <- matrix(0, nrow = length(data), ncol = length(prior))
  dimnames(posterior) <- list(data, names(prior))

  initial_prior <- prior
  for (i in 1:length(data)) {
    posterior[i, ] <- prior*likelihood[, data[i]]/sum(prior*likelihood[, data[i]])
    prior <- posterior[i, ]
  }
  return(rbind(initial_prior, posterior))
}

# 机器正常（绿色线）的后验概率，机器故障（红色线）的后验概率
matplot(bayes(prior, likelihood, data), t = "b", lty = 1, pch = 20, col = c(3, 2))

# 不同的先验概率（结社每种情况指认相同的概率）
prior <- c(working = 0.5, broken = 0.5)
matplot(bayes(prior, likelihood, data), t = "b", lty = 1, pch = 20, col = c(3, 2))

有向概率图模型，可以按照如下形式分解多个随机变量X1,X2,⋅⋅⋅,XnX_1, X_2, ···, X_nX1​,X2​,⋅⋅⋅,Xn​上的联合概率分布：P(X1,X2,⋅⋅⋅,Xn)=∏i=1NP(Xi∣pa(Xi))P(X_1, X_2, ···, X_n) = \prod^{N}_{i = 1} P(X_i | pa(X_i))P(X1​,X2​,⋅⋅⋅,Xn​)=i=1∏N​P(Xi​∣pa(Xi​))，pa(Xi)pa(X_i)pa(Xi​)是图中定义的变量XiX_iXi​的父变量（当箭头从AAA指向BBB时，AAA为BBB的父变量。图是无环的很重要。）的子集。

概率图模型可以使用简单的模块（小的概率分布P(Xi∣pa(Xi))P(X_i | pa(X_i))P(Xi​∣pa(Xi​))）进行构建，并组合出更大的模型。有向模型非常适合建模需要表示因果关系的问题。

无向概率图模型或马尔科夫网络，可以按照如下形式分解多个随机变量X1,X2,⋅⋅⋅,XnX_1, X_2, ···, X_nX1​,X2​,⋅⋅⋅,Xn​上的联合概率分布：P(X1,X2,⋅⋅⋅,Xn)=1Z∏c=1Cψc(χc)P(X_1, X_2, ···, X_n) = \frac{1}{Z} \prod^{C}_{c = 1} \psi_c(\chi_c)P(X1​,X2​,⋅⋅⋅,Xn​)=Z1​c=1∏C​ψc​(χc​)，ZZZ是归一化常数，ψc\psi_cψc​是变量χc\chi_cχc​子集上的因子，以便这个子集的每一个成员是一个极大团，也就是内部所有节点都相互连接的子图。（ψc\psi_cψc​定义在极大团上。）

这种类型的模型在计算机视觉、图像处理、财经和其他变量间关系遵循一定模式的领域都有广泛的应用。

第二章 精确推断

两种类型的查询：

  • 概率查询，观察到变量的一个子集EEE，并选择这些变量的一个实例eee，称为证据。然后计算变量集XXX的子集YYY的后验概率分布：P(Y∣E=e)P(Y | E = e)P(Y∣E=e)
  • MAP查询：EEE为被观察变量的集合，ZZZ为模型的其他变量，MAP赋值可通过MAP=(Z∣E=e)=arg⁡max⁡zP(z,e)MAP = (Z | E = e) = \arg\max_zP(z, e)MAP=(Z∣E=e)=argzmax​P(z,e)来定义。即，我们要找出为被观察的变量的值，满足如果我们观察到赋值E=eE = eE=e，那么未被观察的变量拥有最大的概率。

推断问题，可以化解为通过贝叶斯规则找出后验概率的问题。XXX为模型所有变量集合，EEE为被观察变量（证据）的集合，ZZZ是隐含变量或非观察变量的集合，则计算图模型的推断为：P(Z∣E,θ)=P(Z,E∣θ)P(E∣θ)=P(Z,E∣θ)∑z∈ZP(Z=z,E∣θ)P(Z | E, \theta) = \frac{P(Z, E | \theta)}{P(E | \theta)} = \frac{P(Z, E | \theta)}{\sum_{z \in Z}P(Z = z, E | \theta)}P(Z∣E,θ)=P(E∣θ)P(Z,E∣θ)​=∑z∈Z​P(Z=z,E∣θ)P(Z,E∣θ)​

例如，在医学问题中，给定一个观察到的症状结集合，我们想知道所有可能的疾病。在语音识别系统中，我们想知道被记录的声音中最可能的单词序列。在雷达跟踪系统中，我们想从雷达读数中知道跟踪物体的概率分布。在推荐系统中，在给出售卖网站上用户最近的点击数据后，我们想知道待售产品的后验概率分布，以便给客户提供最优的5个产品的排序和推荐。

构件图模型

图模型的设计考虑两个不同方面，一，确定模型涉及的变量；二，图（表示不同变量之间如何依赖）。连接变量的原因很多，也有许多算法可自动从数据集中学习这些连接，因果关系、稀疏模型、分解···这些原因都足以要求图模型中的变量连接。

另一个生成图模型的方法是模块化，可通过简单模块构建复杂模型，而且可通过扩展图形来扩展已知模型。

变量消解

推断是指，给定模型中变量子集的观测值后，计算其他变量子集的后验概率。解决这个问题通常意味着我们可以选取任一不相交的子集。

设XXX是图模型中所有变量的集合，YYY和EEE是两个不相交的变量子集，Y,E⊂XY, E \subset XY,E⊂X。我们把YYY当作查询子集，也就是需要知道其中变量的后验概率，把EEE作为观测子集，也就是其中的变量都有观测值，也称为证据。得到一个查询的一般形式是：P(Y∣E=e)=P(Y,e)P(e)P(Y | E = e) = \frac{P(Y, e)}{P(e)}P(Y∣E=e)=P(e)P(Y,e)​。事实上，P(Y,e)P(Y, e)P(Y,e)可以看作YYY上的函数，使得P(Y,E=e)→P(y,e)=P(Y=y,E=e)P(Y, E = e) \rightarrow P(y, e) = P(Y = y, E = e)P(Y,E=e)→P(y,e)=P(Y=y,E=e)——即同时有Y=yY = yY=y和E=eE = eE=e的概率。

最后我们可以定义W=X−Y−EW = X - Y - EW=X−Y−E，即图模型中既不是查询变量又不是观测变量的变量子集。然后我们可以计算P(y,e)=∑w∈WP(y,e,w)P(y, e) = \sum_{w \in W}P(y, e, w)P(y,e)=∑w∈W​P(y,e,w)。如果我们沿WWW边缘化，我们只有P(Y,E=e)P(Y, E = e)P(Y,E=e)。进而我们也可以计算证据P(E=e)P(E = e)P(E=e)的概率，例如P(E=e)=∑y∈YP(y,e)P(E = e) = \sum_{y \in Y}P(y, e)P(E=e)=∑y∈Y​P(y,e)。

因此贝叶斯推理的一般机制，是沿着不需要的和观测到的变量进行边缘化，只剩下要查询的变量。例如变量链：A→B→C→DA \rightarrow B \rightarrow C \rightarrow DA→B→C→D，则有P(ABCD)=P(A)⋅P(B∣A)⋅P(C∣B)⋅P(D∣C)P(ABCD) = P(A) \cdot P(B|A) \cdot P(C|B) \cdot P(D|C)P(ABCD)=P(A)⋅P(B∣A)⋅P(C∣B)⋅P(D∣C)。如果P(A)P(A)P(A)可以直接从图中的关联函数中读出，P(B)P(B)P(B)就需要通过沿AAA边缘化计算得出P(B)=∑aP(B∣A=a)P(A=a)P(B) = \sum_aP(B|A = a)P(A = a)P(B)=∑a​P(B∣A=a)P(A=a)。如果A∈RkA \in R^kA∈Rk，B∈RmB \in R^mB∈Rm（即，AAA有kkk个可能的值，BBB有mmm个可能的值。）执行这个求和需要m⋅(2k−1)m \cdot (2k - 1)m⋅(2k−1)次操作：P(B=i)=∑aP(a)P(B=i∣a)P(B = i) = \sum_aP(a)P(B = i|a)P(B=i)=a∑​P(a)P(B=i∣a)P(A=1)P(B=1∣A=1)+⋅⋅⋅+P(A=k)P(B=1∣A=k)P(A=1)P(B=1|A=1)+···+P(A=k)P(B=1|A=k)P(A=1)P(B=1∣A=1)+⋅⋅⋅+P(A=k)P(B=1∣A=k)，需要对BBB的每一个mmm值计算一次上述。

同理可得：P(D)=∑c∑b∑aP(A)⋅P(B∣A)⋅P(C∣B)⋅P(D∣C)P(D)=\sum_c \sum_b \sum_a P(A) \cdot P(B|A) \cdot P(C|B) \cdot P(D|C)P(D)=c∑​b∑​a∑​P(A)⋅P(B∣A)⋅P(C∣B)⋅P(D∣C)P(D)=∑cP(D∣C)∑bP(C∣B)∑aP(A)P(B∣A)P(D) = \sum_c P(D|C) \sum_b P(C|B) \sum_aP(A)P(B|A)P(D)=c∑​P(D∣C)b∑​P(C∣B)a∑​P(A)P(B∣A)。这极大的简化了计算量，因为求和操作只需要使用局部分布。对于给定概率图模型，RnR^nRn中的nnn个变量的链，复杂度只有O(kn)O(k^n)O(kn)。（O(⋅)O(\cdot)O(⋅)表示计算时间的上界与括号中的函数成比例，也称为最差时间复杂度）。

这个例子主要思想，是我们可以对变量求和，并在下一步中重用之前的结果。理想情况下，我们希望对任何图形使用同样的思想，然后通过暂存中间结果来逐步消解变量。这得益于图模型的结构，每一个求和步骤中的表示只依赖少数变量。
